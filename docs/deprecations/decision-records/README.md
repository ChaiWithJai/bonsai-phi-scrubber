# Decision Records

Decision records explain why a pattern is current, deferred, deprecated, or kept
only for reference.

Use one record per decision. Keep it short and operational:

```text
# DR-YYYY-MM-DD: Title

Status:
Decision:
Context:
Current path:
Deprecated/deferred path:
Revival:
Verification:
```
